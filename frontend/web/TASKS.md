# Web 用户端任务清单

## 0. 用户端视觉基调

- [x] 整体配色参考 Apple Mac 页面：https://www.apple.com.cn/mac/
  - 参考方向：浅灰页面背景、白色内容面板、近黑主文字、低饱和灰色辅助文字、蓝色作为主要操作/链接色。
  - 建议色板：页面背景 `#f5f5f7`，内容面 `#ffffff`，主文字 `#1d1d1f`，次级文字 `#6e6e73`，弱分割线 `#d2d2d7`，主行动色 `#0071e3`。
  - 视觉气质：克制、清爽、留白充足，强调内容本身；避免重色块、复杂渐变、过多阴影和装饰性背景。
  - 组件风格：输入框、按钮、结果面板使用细边框和轻阴影，圆角控制在 8px 内；按钮状态清晰但不过度饱和。
  - 字体层级：主标题大而简洁，正文保持 15-16px 高可读性，说明文字使用灰色降级。
  - 使用边界：只参考配色、留白和信息层级，不直接复刻 Apple 页面结构、文案、图形资产或品牌符号。

## 1. 用户端整体布局

- [x] 页面定位为单页规划工作台，首屏直接展示规划输入和生成结果，不做营销式落地页。
- [x] 顶部导航：
  - 高度控制在 48-56px，使用半透明白色或浅灰背景，底部 1px 分割线。
  - 左侧显示产品名"路书匠"，右侧放置历史记录、登录/用户入口；移动端收敛为产品名 + 用户入口。
- [x] 首屏主结构：
  - 桌面端采用左右分栏，最大内容宽度约 1200-1320px，居中展示。
  - 左栏为规划输入面板，宽度约 360-420px，包含起点、目的地、范围、交通方式、日期、人数、偏好和避免项。
  - 右栏为流式输出区域，占据剩余空间，最小高度约 560px，保证生成过程有足够阅读空间。
  - 左右栏间距 24-32px，面板背景使用白色，页面背景使用 `#f5f5f7`。
- [x] 输入面板结构：
  - 顶部为简短标题和一句说明，避免大段介绍。
  - 表单按"去哪儿、怎么去、什么时候、偏好约束"分组，组标题使用小号灰色文字。
  - 交通方式使用分段选择或图标按钮；偏好和避免项使用可多选标签，不使用大面积彩色块。
  - 生成按钮固定在面板底部或表单末尾，使用主行动蓝 `#0071e3`，加载中明确禁用。
- [x] 结果区结构：
  - 默认空态展示简洁引导和最近一次规划入口，不堆叠说明文案。
  - 生成中优先显示流式输出、阶段进度和停止操作。
  - 生成完成后在结果顶部保留摘要，下面依次展示天气预警、路线/公共交通、高德路径图、途径景点、实时信息检索、最终 Markdown。
- [x] 下方内容区：
  - 结果完成后在主区域下方展示历史生成记录列表，桌面端可做两列紧凑列表，移动端单列。
  - 历史记录卡片只展示起终点、交通方式、状态、生成时间和进入详情操作。
- [x] 移动端结构：
  - 采用单列布局，顺序为顶部导航、规划输入、生成按钮、流式输出、结果模块、历史记录。
  - 输入面板不使用固定定位，避免遮挡移动端键盘；底部可保留轻量吸底生成按钮。
  - 所有主要按钮触控高度不低于 44px，表单项纵向间距 12-16px。
- [x] 移动端底部 Tab：
  - 移动端使用底部 Tab 切换，桌面端不使用底部 Tab。
  - Tab 建议为 4 个：首页/规划、结果、历史、我的。
  - 底部 Tab 高度约 56-64px，背景使用半透明白色或 `#ffffff`，顶部 1px 分割线，适配安全区 `safe-area-inset-bottom`。
  - 当前 Tab 使用主行动蓝 `#0071e3`，未选中使用次级灰 `#6e6e73`；图标 + 短文字组合，文字不超过 2-3 个字。
  - 生成中时"结果"Tab 显示进行中状态点；生成失败时显示轻量错误提示点。
  - 底部 Tab 不遮挡内容，页面底部需要预留与 Tab 等高的 padding。
- [x] 交互反馈：
  - 表单校验错误显示在字段下方，不使用弹窗打断输入。
  - 生成中页面不跳转，输出区内部滚动；用户手动滚动查看上文时不要强制拉回底部。
  - 重新生成需要保留上一版结果直到新结果开始输出，避免页面瞬间空白。

## 2. 用户端功能

- [x] 首页即规划页。
- [x] 起点输入。
- [x] 目的地输入。
- [x] 范围输入。
- [x] 交通方式选择。
- [x] 日期、人数、偏好和避免项。
- [x] 生成按钮。
- [x] 流式输出区域。
  - 样式定位：作为规划页的主阅读区，优先保证长文本可读、阶段状态清晰、用户能随时停止/复制/保存结果。
  - 布局结构：顶部固定为生成状态栏，包含状态点、当前阶段、耗时、停止生成、复制结果等操作；中部为流式正文；底部保留完成后操作区。
  - 阶段呈现：用横向或纵向步骤条展示"需求理解、天气、路线、景点、实时信息、汇总"，当前阶段高亮，已完成阶段显示轻量完成态。
  - 正文样式：白色或浅灰底内容面板，1px 细边框，圆角不超过 8px，正文 15-16px，行高 1.7，段落之间保持 10-12px 间距。
  - 流式动效：新增文本使用轻微淡入，末尾显示细小闪烁光标；避免整块内容重排或大幅滚动跳动。
  - 信息分层：天气预警、路线建议、途径景点、实时信息检索用紧凑分组标题和浅色标签区分，来源和更新时间放在次级灰色文本中。
  - 状态覆盖：需要包含生成中、暂停/取消中、生成完成、生成失败、无内容、重新生成中的视觉状态。
  - 响应式：桌面端可与输入表单左右分栏，移动端输出区域置于表单下方；操作按钮最小触控高度 44px，正文不横向溢出。
- [x] 天气预警模块。
- [x] 路径点/公共交通规划模块。
- [x] 高德路径图预览。
- [x] 高德路线链接打开按钮。
- [x] 路径图保存或下载入口。
- [x] 途径景点模块。
- [x] 实时信息检索模块。
  - 分为"新闻/交通管制"和"攻略/避坑参考"两个分组。
  - 每条信息展示标题、摘要、来源站点、发布时间、原文链接和分类标签。
  - 攻略/避坑参考需要标注"用户经验仅供参考，出行前请复核"。
- [x] 最终结果 Markdown 渲染。
- [x] 历史生成记录列表。
- [x] 历史生成记录详情。
- [ ] 对齐后端 SSE `snapshot.type` 契约。负责人：FE-Web、BE-Stream；状态：进行中；优先级：P0；说明：当前前端监听 `route/realtime/attractions`，后端 Mock 发 `routes/realtime_info/summary`，会导致部分模块不展示。
