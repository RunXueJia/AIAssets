# 需求评估

基于 `PROJECT_REQUIREMENTS.md` v1.0，项目定位清晰：建设一个面向外部增长的 AI 内容资产生产系统，核心价值不是自动发视频，而是稳定沉淀短视频、图文页、资料包、线索和运营数据。

## 1. 需求结论

项目适合拆为两个产品面：

- 公开站：承接 SEO、内容浏览、资料下载和线索收集。
- 管理后台：配置内容方向、生产内容、审核、渲染、导出发布包、查看数据。

项目适合拆为六个后端能力域：

- 内容配置域：内容方向、栏目、模板、平台限制。
- LLM 调用域：模型配置、提示词模板、OpenAI 标准 SSE 流式输出、结构化输出、调用日志、成本与限流。
- 内容生产域：选题、脚本、分镜、图文页、知识卡片、资料包。
- 审核发布域：审核流、版本记录、发布队列、发布包导出。
- 自动化任务域：定时调度、异步队列、重试、日志、日报。
- 增长数据域：访问、下载、线索、转化、看板。

MVP 应优先跑通从「配置栏目 -> 生成选题 -> 生成脚本 -> 生成分镜 -> 审核 -> 视频合成 -> 图文页 -> 发布包 -> 数据看板」的闭环。

## 2. 范围评估

### MVP 必做

- 后台登录与基础 RBAC。
- LLM 模型配置、提示词模板、OpenAI 标准 SSE 流式调用、调用日志和结构化输出校验。
- 内容方向和栏目配置。
- 手动触发与定时触发的内容生产任务。
- 选题、脚本、分镜、图文页生成。
- 审核通过、驳回、修改后通过、重新生成。
- 已审核内容进入视频合成队列。
- 视频、封面、图文页、发布素材包管理。
- 公开站内容浏览、资料下载、表单线索。
- 基础数据看板和每日报告。

### MVP 可降级

- 视频合成先使用固定模板与 FFmpeg/HTML 渲染，不做复杂编辑器。
- 发布渠道先做导出包和平台链接记录，不直连平台 API。
- 知识卡片先做固定品牌样式导出 PNG。
- 资料包先支持 PDF、Markdown、Prompt 模板三类。
- 数据分析先做站内访问、下载、线索、内容产出，不追踪平台播放数据。

### 暂缓到 v1.0 后

- 多租户。
- 平台 API 自动发布。
- 高级 SEO 内链策略。
- A/B 测试。
- 数字人。
- 多语言。
- CRM 或企业内部系统集成。

## 3. 关键风险

| 风险 | 影响 | 建议 |
| --- | --- | --- |
| 内容生产链路长 | 任一环节失败会影响产出 | 每一步独立任务化，记录状态、错误和重试次数 |
| LLM 流式输出不完整 | SSE 中断会导致 JSON 不完整 | 保存 chunk、断流标记失败、支持从任务重新生成，不把不完整结果写入正式内容 |
| LLM 输出不可控 | JSON 解析失败或内容偏离要求 | 使用固定 Prompt 模板、OpenAI 标准 SSE、JSON Schema 校验、失败修复和人工审核 |
| LLM 成本不可控 | 调用费用超预算 | 记录 token、耗时、模型、任务来源，设置每日额度和并发限制 |
| AI 内容质量不稳定 | 审核成本升高 | 建立风险词、事实检查、重复检测和人工审核 |
| 视频渲染耗时高 | 队列堆积 | 审核通过后再渲染，限制并发和每日上限 |
| 状态流复杂 | 前后端联调成本高 | 统一内容状态机，所有状态变更走后端接口 |
| 公开站和后台权限混淆 | 数据泄露 | 公开接口与后台接口分组，后台强制鉴权 |
| 文件存储增长快 | 运维成本上升 | 文件统一对象存储，保存元数据和清理策略 |

## 4. 状态机建议

### 选题状态

- `generated`：已生成
- `locked`：已锁定
- `rejected`：已驳回
- `script_ready`：脚本已生成

### 脚本状态

- `draft`：草稿
- `pending_review`：待审核
- `approved`：审核通过
- `rejected`：已驳回
- `regenerating`：重新生成中

### 视频状态

- `waiting`：待合成
- `rendering`：合成中
- `success`：合成成功
- `failed`：合成失败

### 发布状态

- `draft`：草稿
- `pending_review`：待审核
- `approved`：审核通过
- `rendering`：合成中
- `render_failed`：合成失败
- `pending_publish`：待发布
- `published`：已发布
- `offline`：已下线

## 5. 推荐交付顺序

1. 搭建 Python FastAPI 后端、Vue3 管理后台、Vue3 公开站基础工程。
2. 实现用户登录、角色权限、统一接口协议。
3. 实现 LLM 调用域：模型配置、提示词模板、OpenAI 标准 SSE 流式调用、调用日志、结构化输出校验。
4. 实现内容方向、栏目、模板、平台等基础配置。
5. 实现内容生产数据模型和人工触发生成接口。
6. 实现审核流和版本记录。
7. 接入异步队列、定时任务、任务日志和失败重试。
8. 实现视频合成、文件存储、发布包导出。
9. 实现公开内容页、资料下载、线索收集。
10. 实现数据看板和日报。
11. 做 7 天连续运行验证和验收数据整理。
