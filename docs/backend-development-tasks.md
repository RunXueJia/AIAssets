# 后端开发任务列表

技术栈建议：Python 3.12、FastAPI、SQLAlchemy 2、Alembic、PostgreSQL、Redis、Celery 或 RQ、APScheduler、MinIO 或 S3 兼容对象存储。

## 1. 基础工程

- [x] 初始化 `services/api` FastAPI 项目。
- [x] 初始化 `services/worker` 异步任务项目。
- [x] 初始化 `services/renderer` 视频渲染服务。
- [x] 建立统一配置管理：数据库、Redis、对象存储、AI 模型、JWT、CORS。
- [x] 建立统一响应格式：`code`、`message`、`data`。
- [x] 建立异常处理：参数错误、认证失败、权限不足、资源不存在、业务异常、系统异常。
- [x] 建立日志规范：`trace_id`、`user_id`、`task_id`、`content_id`、耗时、错误堆栈。
- [x] 建立 Alembic 迁移流程。
- [x] 建立基础测试框架：pytest、httpx AsyncClient、测试数据库。

- [x] 初始化权限点：配置管理、内容审核、视频渲染、发布队列、线索导出、数据查看、系统管理。
- [x] 实现用户列表、创建、编辑、禁用、重置密码。
- [x] 实现角色权限分配。
- [ ] 实现权限变更审计日志。

## 3. 内容配置

## 4. 内容生产链路

- [x] 实现 `Topic` 选题模型和状态流。
- [x] 实现选题生成 Service：组装业务上下文，调用 LLM 网关，根据内容方向、栏目、历史选题、每日额度生成候选选题。
- [ ] 实现选题去重：标题相似度、关键词重叠、同栏目最近内容。
- [x] 实现选题审核动作：锁定、驳回、生成脚本。
- [x] 实现 `Script` 脚本模型，支持 15 秒、60 秒、180 秒版本。
- [x] 实现脚本生成 Service：调用 LLM 网关输出钩子、场景、方法、步骤、示例、结尾、平台文案和风险标记。
- [x] 实现脚本版本记录：每次修改或重新生成保存版本。
- [x] 实现 `Storyboard` 分镜模型。
- [x] 实现分镜生成 Service：调用 LLM 网关控制镜头数量、总时长误差、字幕和画面建议。
- [ ] 实现内容质量检查：规则引擎 + LLM 审查，标记敏感词、夸张承诺、高风险主题、可疑事实。

## 12. LLM 调用与提示词

- [x] 建立 `LLMProvider` 抽象：OpenAI 兼容接口、本地模型接口、后续其他供应商接口。
- [ ] 实现 `LLMGatewayService`：统一封装 OpenAI Chat Completions 兼容 SSE 流式调用、结构化输出、重试、超时、降级和日志。
- [x] 规定所有正式 LLM 调用必须使用 `stream: true`，禁止业务生成链路使用非流式 LLM 调用。
- [x] 实现 SSE 上游适配：读取供应商 `data: {chat.completion.chunk}` 和 `data: [DONE]`。
- [x] 实现 SSE 下游转发：FastAPI 使用 `StreamingResponse` 返回 `text/event-stream`，保持 OpenAI chunk 格式。
- [x] 实现流式聚合器：实时转发 chunk，同时累积 `delta.content` 用于最终 JSON 解析。
- [x] 实现流式中断处理：客户端断开、上游超时、未收到 `[DONE]` 都标记为失败，不写入正式内容。
- [x] 设计 `ModelProvider` 模型供应商表：名称、base_url、api_key 密文、状态、默认参数。
- [x] 设计 `ModelConfig` 模型配置表：模型名、用途、上下文长度、温度、最大输出 token、成本单价、状态。
- [x] 设计 `PromptTemplate` 提示词模板表：场景、版本、system prompt、user prompt、JSON Schema、状态。
- [x] 设计 `LLMCallLog` 调用日志表：任务 ID、业务对象、模型、模板版本、输入输出 token、首 token 耗时、总耗时、费用估算、状态、错误。
- [x] 设计 `LLMStreamChunk` 流式分片表或日志字段：序号、chunk JSON、delta 文本、到达时间、是否完成。
- [ ] 设计 `LLMOutputVersion` 输出版本表：流式聚合文本、解析后 JSON、校验结果、修复记录、关联内容版本。
- [ ] 建立提示词模板目录：选题、脚本、分镜、质检、图文页、知识卡片、资料包、日报建议。
- [x] 实现 Prompt 渲染：把栏目配置、内容方向、历史内容、平台限制、安全规则注入模板。
- [ ] 实现 JSON Schema 校验：失败时先自动修复一次，再失败则标记任务失败并保留原始响应。
- [ ] 实现 LLM 调用限流：按供应商、模型、任务类型、每日预算控制并发和调用次数。
- [ ] 实现 LLM 调用重试：网络错误、限流错误、结构化输出错误分别处理。
- [x] 实现模型密钥后端加密保存，接口禁止返回明文。
- [ ] 实现调用成本统计：按日期、模型、任务类型、栏目聚合。
- [ ] 实现模型健康检查：连通性、可用模型列表、最近错误率。
- [x] 实现人工测试调用：后台输入 prompt 变量后通过 SSE 实时返回 OpenAI chunk，并在完成后返回解析结果，不写入正式内容。
- [ ] 实现内容安全二次检查：对 LLM 输出执行敏感词、风险主题、事实不确定提示。
