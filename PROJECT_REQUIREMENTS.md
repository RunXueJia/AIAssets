## 1. 项目定位

建设一个后台系统，用于自动生成 AI 科普短视频内容素材。运营人员在后台输入内容方向或具体主题，系统调用 LLM 生成选题、脚本、分镜、字幕、封面文案、知识卡片文案和资料包草稿，并通过 SSE 流式输出生成过程。

当前阶段只做后台功能，不做公开前台网站，不做公开内容页，不做 SEO，不做用户订阅和线索收集。外部发布由运营人员下载发布包后，手动发布到抖音、小红书、B 站、视频号等平台。

本项目不接入公司内部 CRM、ERP、OA、财务、人事等系统，不做内部监控、舆情监控、竞品监控或风险预警。

## 2. 核心目标

1. 后台输入方向：运营输入内容方向、目标受众或具体主题。
2. LLM 自动生成：系统调用 LLM 生成选题、脚本、分镜和素材文案。
3. SSE 流式输出：生成过程在后台页面实时显示，不需要等待整段结果一次性返回。
4. 人工审核：内容必须经过后台审核后才能进入视频合成和发布包导出。
5. 视频合成：审核通过后合成竖屏短视频、封面和知识卡片。
6. 发布包导出：后台导出视频、封面、标题、简介、标签和文案。
7. 数据记录：记录生成任务、审核状态、合成状态、发布链接和产出日报。

## 3. 用户角色

- 管理员：维护账号、模型密钥和基础系统参数。
- 运营负责人：输入内容方向，确认每日目标数量，查看产出数据。
- 内容编辑：审核和修改选题、脚本、分镜、字幕和文案。
- 视频运营：导出发布包，手动上传平台，回填外部发布链接。
- 只读查看者：查看任务状态、内容列表和日报。

MVP 只做简单角色权限，不做复杂权限矩阵。

## 4. 内容方向

第一阶段聚焦“AI 工具 + 职场效率科普”。

系统内置以下栏目，MVP 默认启用前 3 个，不需要复杂配置：

1. 《一分钟 AI 办公》：60 秒讲一个具体办公技巧。
2. 《今天少加班一小时》：讲 AI 如何替代重复劳动。
3. 《老板也能听懂的 AI》：面向小微企业主讲业务场景。
4. 《普通人 AI 工具箱》：介绍一个工具和一个可执行场景。
5. 《AI 避坑指南》：讲清常见误区、风险边界和错误用法。

运营只需要输入：

- 内容方向，例如“AI 工具 + 职场效率”。
- 可选主题，例如“AI 写周报”。
- 可选目标受众，例如“普通职场人”。
- 今日目标数量。
- 是否立即生成。

其他参数使用系统默认值。

## 5. 核心流程

流程尽量固定，减少人工配置：

1. 运营在后台输入内容方向、主题或目标受众。
2. 后台发起生成任务，请求 API 创建任务记录。
3. API 调用 LLM，使用 SSE 将生成过程实时推送到前端。
4. LLM 先生成候选选题，再生成脚本、分镜、字幕和封面文案。
5. 系统执行基础质量检查，标记重复、风险词、夸张承诺和事实疑点。
6. 生成结果进入审核列表。
7. 编辑可通过、修改、驳回或重新生成。
8. 审核通过后，系统合成竖屏视频、封面和知识卡片。
9. 视频运营导出发布包，手动发布到外部平台。
10. 视频运营回填平台链接和发布状态。
11. 系统记录任务状态、审核状态、合成状态和每日产出。

MVP 不做复杂流程编排，不做多级审核，不做细粒度规则配置。

### 6.2 内容生成入口

后台提供一个简单生成表单。

字段：

- 内容方向，必填
- 具体主题，可选
- 目标受众，可选
- 生成数量，默认 5
- 栏目，默认自动匹配
- 生成类型，默认“选题 + 脚本 + 分镜”

不做复杂配置项，不做提示词规则后台，不做可视化工作流。

### 6.3 LLM 生成与 SSE 流式输出

系统调用 LLM 生成内容，前端通过 SSE 实时接收生成结果。

生成内容包括：

- 候选选题
- 平台标题
- 60 秒短视频脚本
- 分镜
- 字幕
- 封面文案
- 视频简介
- 平台标签
- 知识卡片文案
- 资料包草稿

SSE 输出要求：

- 使用 `text/event-stream`。
- 生成开始时推送任务信息。
- 生成过程中持续推送文本增量和阶段状态。
- 每完成一个阶段推送结构化结果。
- 出错时推送错误信息并记录任务日志。
- 完成时推送最终内容 ID 和任务状态。

建议事件类型：

- `start`：任务开始
- `stage`：当前生成阶段
- `delta`：LLM 文本增量
- `result`：阶段结构化结果
- `error`：生成失败
- `done`：生成完成

### 6.4 选题管理

系统根据输入方向和内置栏目生成候选选题。

选题包含：

- 标题
- 目标受众
- 内容角度
- 推荐栏目
- 推荐时长
- 关键词
- 生成原因
- 状态

验收要求：

- 支持批量生成候选选题。
- 支持通过、驳回、锁定和重新生成。
- 选题可继续生成脚本和分镜。

### 6.5 脚本生成

系统为选题生成 60 秒短视频脚本。

脚本结构：

- 开头钩子
- 用户痛点
- 核心方法
- 操作步骤
- 示例
- 总结
- 结尾引导
- 平台标题、简介和标签

验收要求：

- 脚本可直接拆分为字幕和分镜。
- 生成过程支持 SSE 流式显示。
- 自动标记敏感表述、夸张承诺和事实疑点。

### 6.6 分镜与字幕生成

系统将脚本拆成分镜和字幕。

分镜包含：

- 镜头序号
- 时长
- 旁白
- 屏幕字幕
- 画面类型
- 素材建议
- 动效建议

验收要求：

- 60 秒视频生成 6 到 10 个分镜。
- 分镜总时长接近目标时长。
- 每个分镜必须包含字幕和画面建议。

### 6.7 审核后台

所有生成内容进入同一个审核列表。

审核动作：

- 通过
- 修改后通过
- 驳回
- 重新生成

审核规则：

- 未审核内容不能合成视频。
- 驳回内容必须填写原因。
- 修改后通过需要保留版本记录。
- 涉及不确定事实必须人工确认。
- 涉及高风险表达必须改写或删除。

### 6.8 视频合成

审核通过后，系统根据分镜和模板合成竖屏短视频。

视频规格：

- 比例：9:16
- 分辨率：1080x1920
- 格式：MP4
- 帧率：30fps
- 字幕：硬字幕

验收要求：

- 审核通过后可进入合成队列。
- 合成失败需要记录原因并支持重试。
- 合成后可在后台预览和下载。
- 字幕和标题不能超出画面边界。

### 6.9 发布包导出

发布包包含：

- 视频文件
- 封面图

## 9. 技术架构

推荐模块：

1. Admin 后台：生成入口、SSE 展示、审核、合成队列、发布包和数据看板。
2. API 服务：提供业务接口、鉴权、SSE 接口和任务管理。
3. LLM 生成服务：调用大模型生成选题、脚本、分镜和文案。
4. Worker 服务：执行队列任务、质量检查、失败重试和日报生成。
5. Renderer 服务：合成视频、封面和知识卡片。
6. 数据服务：保存内容、任务、资产、审核记录和发布记录。
7. 文件存储：保存视频、封面、卡片、资料包和发布包。

推荐技术选型：

- 后台前端：Vue Admin 或 Next.js Admin
- 后端：FastAPI 或 NestJS
- 数据库：PostgreSQL
- 队列：Redis + BullMQ 或 Celery
- SSE：原生 EventSource 或 Fetch ReadableStream
- 定时任务：Cron 或 APScheduler
- 文件存储：MinIO 或本地存储目录
- 视频合成：FFmpeg + HTML/CSS 模板渲染
- LLM：OpenAI 兼容接口或可替换模型网关

## 10. SSE 接口建议

### 10.1 创建生成任务

```text
POST /api/generation/tasks
```

请求参数：

- direction：内容方向
- topic：具体主题，可选
- audience：目标受众，可选
- count：生成数量
- generation_type：生成类型

返回：

- task_id
- status

### 10.2 订阅生成流

```text
GET /api/generation/tasks/{task_id}/stream
```

响应类型：

```text
Content-Type: text/event-stream
```

事件示例：

```text
event: stage
data: {"stage":"生成脚本","task_id":"xxx"}

event: delta
data: {"text":"今天教你用 AI 快速生成周报..."}

event: result
data: {"type":"script","content_id":"xxx"}

event: done
data: {"task_id":"xxx","status":"success"}
```

### 10.3 查询任务结果

```text
GET /api/generation/tasks/{task_id}
```

用于页面刷新后恢复任务状态和最终结果。

## 13. 非功能要求

### 稳定性

- 系统支持长时间运行。
- 单个任务失败不能阻塞队列。
- 任务支持失败重试。
- 自动任务必须记录日志。
- 生成任务可以取消或重新生成。
- 页面刷新后可以恢复任务状态。

### SSE 可靠性

- SSE 断开后，前端可以重新连接并查询任务结果。
- 生成过程中的阶段结果需要落库。
- LLM 原始输出和结构化结果需要保留。
- 错误信息需要返回给前端并写入任务日志。

### 安全性

- API 密钥不得暴露在前端。
- 后台必须登录访问。
- 表单输入需要做长度限制和基础校验。
- 删除和发布状态变更需要记录操作者。

### 内容安全

- 不生成违法、歧视、侵权、低俗内容。
- 不输出医疗、金融、法律等高风险定论型建议。
- 不做虚假承诺或夸张收益表达。
- 生成内容必须经过人工审核后才允许合成和导出。

### 可扩展性

- LLM 调用层可替换。
- 视频模板可新增。
- 栏目模板可新增。
- 发布渠道可扩展。
- 后续可再增加公开前台，但不属于当前阶段。

## 19. CEO 汇报口径

当前阶段先建设后台内容生产系统，不做公开前台网站。运营人员在后台输入内容方向或主题，系统调用 LLM 自动生成选题、脚本、分镜、字幕和素材文案，并通过 SSE 实时展示生成过程。

项目价值先通过后台生产效率验证：每天能生成多少选题、脚本和视频素材，审核通过率如何，视频合成是否稳定，发布包是否能支撑运营人员手动发布。公开网站、订阅、线索收集和 SEO 可作为后续阶段扩展。
