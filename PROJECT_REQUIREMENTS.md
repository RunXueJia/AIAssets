# 24小时AI增长资产引擎需求简版

版本：v1.1  
日期：2026-05-19  
目录：E:\code\hours24  
项目名称：24小时AI增长资产引擎  
项目类型：后台内容生产系统 / 网络素材抓取 / LLM 脚本分镜生成 / 短视频素材管理

## 1. 项目定位

建设一个面向非开发人员使用的后台系统，用于自动生成 AI 科普短视频内容素材。运营人员在后台用自然语言输入内容方向或具体主题，系统先从网络抓取相关内容并形成素材汇总，再调用 LLM 总结生成选题、脚本、分镜、字幕、封面文案、知识卡片文案和资料包草稿，并通过 SSE 流式输出生成过程。

当前阶段只做后台功能，不做公开前台网站，不做公开内容页，不做 SEO，不做用户订阅和线索收集。外部发布由运营人员下载发布包后，手动发布到抖音、小红书、B 站、视频号等平台。

## 2. 核心目标

1. 后台输入方向：运营用业务语言输入内容方向、目标受众或具体主题。
2. 网络素材抓取：系统按任务方向从网络抓取相关内容，形成素材汇总和来源列表。
3. LLM 总结生成：系统调用 LLM 基于抓取内容生成选题、脚本、分镜和素材文案。
4. SSE 流式输出：抓取、汇总和生成过程在后台页面实时显示。
5. 人工审核：内容必须经过后台审核后才能进入视频合成和发布包导出。
6. 视频合成：审核通过后合成竖屏短视频、封面和知识卡片。
7. 自动任务：支持为某个话题开启定时抓取和自动汇总。
8. 发布包导出：后台导出视频、封面、标题、简介、标签和文案。
9. 数据记录：记录抓取任务、生成任务、审核状态、合成状态、发布链接和产出日报。

## 3. 用户角色

- 管理员：维护账号、模型密钥和少量基础系统设置。
- 运营负责人：输入内容方向，确认每日目标数量，查看产出数据。
- 内容编辑：审核和修改选题、脚本、分镜、字幕和文案。
- 视频运营：导出发布包，手动上传平台，回填外部发布链接。
- 只读查看者：查看任务状态、内容列表和日报。

MVP 只做简单角色权限，不做复杂权限矩阵。

## 4. 内容方向

第一阶段聚焦“AI 工具 + 职场效率科普”。

系统内置以下栏目，MVP 默认启用前 3 个，不需要复杂配置：

1. 《一分钟 AI 办公》：60 秒讲一个具体办公技巧。
2. 《今天少加班一小时》：讲 AI 如何替代重复劳动。
3. 《老板也能听懂的 AI》：面向小微企业主讲业务场景。
4. 《普通人 AI 工具箱》：介绍一个工具和一个可执行场景。
5. 《AI 避坑指南》：讲清常见误区、风险边界和错误用法。

运营只需要填写能直接理解的内容：

- 我想做什么内容，例如“AI 工具 + 职场效率”。
- 这次想讲什么主题，例如“AI 写周报”。
- 主要给谁看，例如“普通职场人”。
- 这次生成几条，默认 5 条。
- 是否现在开始生成。

其他技术参数使用系统默认值，不展示给运营人员。

## 5. 核心流程

流程尽量固定，减少人工配置：

1. 运营在后台用自然语言填写内容方向、主题或目标受众。
2. 系统创建生成任务，并记录任务方向、主题、目标受众和生成数量。
3. Worker 执行任务，从网络抓取与话题相关的公开内容。
4. 系统将抓取内容整理为素材汇总和来源列表。
5. API 调用 LLM，基于素材汇总生成重点内容。
6. LLM 总结生成候选选题、60 秒脚本、分镜、字幕、封面文案、平台标题、简介和标签。
7. SSE 将抓取进度、汇总结果和 LLM 生成过程实时推送到后台页面。
8. 生成结果进入审核列表。
9. 编辑可通过、修改后通过、驳回或重新生成。
10. 审核通过后，系统进入视频合成流程，生成竖屏视频、封面和知识卡片。
11. 视频运营导出发布包，手动发布到外部平台。
12. 视频运营回填平台链接和发布状态。
13. 系统记录抓取状态、生成状态、审核状态、合成状态和每日产出。

自动任务流程：

1. 运营为某个话题开启监控，例如“AI 办公工具更新”。
2. 系统按设定频率每天自动抓取相关公开内容。
3. 系统在该话题任务下形成每日素材汇总和内容列表。
4. 运营可从汇总列表中选择内容，一键生成脚本、分镜和视频素材。
5. 自动任务只负责抓取和汇总，进入发布前仍需人工审核。

MVP 不做复杂流程编排，不做多级审核，不做细粒度规则配置。后台页面不直接展示技术字段名、模型参数、提示词变量或接口参数。

### 6.3 LLM 生成与 SSE 流式输出

系统基于网络抓取内容调用 LLM 生成内容，前端通过 SSE 实时接收抓取、汇总和生成结果。

生成内容包括：

- 相关内容素材汇总
- 来源列表
- 候选选题
- 平台标题
- 60 秒短视频脚本
- 分镜
- 字幕
- 封面文案
- 视频简介
- 平台标签
- 知识卡片文案
- 资料包草稿

SSE 输出要求：

- 使用 `text/event-stream`。
- 生成开始时推送任务信息。
- 抓取过程中推送抓取阶段和已发现的内容数量。
- 汇总过程中推送素材摘要和来源信息。
- 生成过程中持续推送文本增量和阶段状态。
- 每完成一个阶段推送结构化结果。
- 出错时推送错误信息并记录任务日志。
- 完成时推送最终内容 ID 和任务状态。

建议事件类型：

- `start`：任务开始
- `stage`：当前生成阶段
- `source`：抓取到的来源信息
- `delta`：LLM 文本增量
- `result`：阶段结构化结果
- `error`：生成失败
- `done`：生成完成

### 6.4 网络抓取与素材汇总

系统根据运营输入的内容方向、主题和目标受众，从网络抓取相关公开内容，并形成可审核、可追溯的素材汇总。

抓取内容包括：

- 标题
- 来源网站
- 来源链接
- 发布时间
- 摘要
- 与当前话题的相关性说明
- 可用于脚本的重点信息

验收要求：

- 每次生成任务先形成素材汇总，再调用 LLM 生成脚本和分镜。
- 每条素材需要保留来源链接。
- 抓取失败不能导致整个任务静默失败，需要给出可理解提示。
- LLM 生成内容需要基于素材汇总，不直接脱离素材编造事实。
- 涉及不确定信息时，需要在审核列表中标记为“需要人工确认”。

### 6.5 话题监控与自动任务

运营可以为某个话题开启监控。开启后，系统按固定时间自动抓取相关内容，并在该话题下形成每日汇总和内容列表。

页面字段：

- 监控的话题  
  示例：“AI 办公工具更新”“AI 视频生成工具”“企业 AI 培训案例”

- 主要关注的人群  
  示例：“普通职场人”“小微企业老板”“内容运营人员”

- 每天什么时候更新  
  默认每天上午 9 点。

- 每次最多抓取多少条  
  默认 20 条。

- 是否自动生成选题  
  默认开启，只生成选题，不自动合成视频。

自动任务规则：

- 自动任务可以暂停、恢复和删除。
- 每天抓取结果保存在对应话题下。
- 每天形成一份素材汇总和来源列表。
- 系统可基于每日汇总自动生成候选选题。
- 自动任务生成的内容仍需人工审核后才能合成视频。

系统面向非开发人员使用，后台配置必须语义化。

页面上应该出现的表达：

- “你想做什么内容”
- “这次想讲什么主题”
- “主要给谁看”
- “一次生成几条”
- “这次要生成到哪一步”
- “内容更适合哪个栏目”

页面上不应该出现的表达：

- `direction`
- `topic`
- `audience`
- `count`
- `generation_type`
- `prompt`
- `temperature`
- `top_p`
- `max_tokens`
- `model`

技术参数处理规则：

- 栏目用中文栏目名呈现，默认由系统自动判断。
- 错误提示必须给出可执行建议，不直接展示接口异常、堆栈或模型原始错误。
- 任务 ID、接口参数、原始提示词和模型返回原文只在管理员日志中可见。

### 6.9 分镜与字幕生成

系统将脚本拆成分镜和字幕。

分镜包含：

- 镜头序号
- 时长
- 旁白
- 屏幕字幕
- 画面类型
- 素材建议
- 动效建议

验收要求：

- 60 秒视频生成 6 到 10 个分镜。
- 分镜总时长接近目标时长。
- 每个分镜必须包含字幕和画面建议。

### 6.10 审核后台

所有生成内容进入同一个审核列表。

审核动作：

- 通过
- 修改后通过
- 驳回
- 重新生成

审核规则：

- 未审核内容不能合成视频。
- 驳回内容必须填写原因。
- 修改后通过需要保留版本记录。
- 涉及不确定事实必须人工确认。
- 涉及高风险表达必须改写或删除。

### 6.11 视频合成

审核通过后，系统根据分镜和模板合成竖屏短视频。

视频规格：

- 比例：9:16
- 分辨率：1080x1920
- 格式：MP4
- 帧率：30fps
- 字幕：硬字幕

验收要求：

- 审核通过后可进入合成队列。
- 合成失败需要记录原因并支持重试。
- 合成后可在后台预览和下载。
- 字幕和标题不能超出画面边界。

## 9. 技术架构

推荐模块：

1. Admin 后台：生成入口、SSE 展示、审核、合成队列、发布包和数据看板。
2. API 服务：提供业务接口、鉴权、SSE 接口和任务管理。
3. 抓取服务：根据话题抓取公开网络内容并保存来源。
4. LLM 生成服务：基于素材汇总调用大模型生成选题、脚本、分镜和文案。
5. Worker 服务：执行抓取任务、自动任务、质量检查、失败重试和日报生成。
6. Renderer 服务：合成视频、封面和知识卡片。
7. 数据服务：保存内容、素材、任务、资产、审核记录和发布记录。
8. 文件存储：保存视频、封面、卡片、资料包和发布包。

推荐技术选型：

- 后台前端：Vue Admin 或 Next.js Admin
- 后端：FastAPI 或 NestJS
- 数据库：PostgreSQL
- 队列：Redis + BullMQ 或 Celery
- SSE：原生 EventSource 或 Fetch ReadableStream
- 定时任务：Cron 或 APScheduler
- 网络抓取：HTTP 客户端 + 页面解析器，必要时可接搜索 API
- 文件存储：MinIO 或本地存储目录
- 视频合成：FFmpeg + HTML/CSS 模板渲染
- LLM：OpenAI 兼容接口或可替换模型网关

## 10. SSE 接口建议

### 10.1 创建生成任务

```text
POST /api/generation/tasks
```

后台页面使用中文业务字段，提交到 API 前由前端或后端转换为内部参数。

页面字段到内部参数的映射：

- 你想做什么内容 -> `direction`
- 这次想讲什么主题 -> `topic`
- 主要给谁看 -> `audience`
- 一次生成几条 -> `count`
- 这次要生成到哪一步 -> `generation_type`

内部请求参数：

- direction：内容方向
- topic：具体主题，可选
- audience：目标受众，可选
- count：生成数量
- generation_type：生成类型
- enable_fetch：是否先抓取网络素材，默认开启

要求：

- 页面不展示内部参数名。
- 接口错误需要转换成运营可理解的提示。
- 管理员日志可以查看内部参数，普通运营界面不展示。

返回：

- task_id
- status

### 10.2 创建话题监控任务

```text
POST /api/monitors
```

页面字段到内部参数的映射：

- 监控的话题 -> `topic`
- 主要关注的人群 -> `audience`
- 每天什么时候更新 -> `schedule_time`
- 每次最多抓取多少条 -> `fetch_limit`
- 是否自动生成选题 -> `auto_generate_topics`

要求：

- 页面不展示 cron 表达式。
- 运营只选择“每天几点更新”。
- 自动任务默认只生成素材汇总和候选选题，不自动合成视频。

返回：

- monitor_id
- status

## 12. 权限要求

- 后台所有页面必须登录访问。
- 后台所有接口必须鉴权。
- 管理员可维护用户、模型密钥和少量基础参数。
- 运营负责人可创建生成任务、查看发布计划和数据。
- 内容编辑可审核和修改内容。
- 视频运营可导出发布包并记录发布链接。
- 只读查看者只能查看列表和报表。

## 13. 非功能要求

### 稳定性

- 系统支持长时间运行。
- 单个任务失败不能阻塞队列。
- 任务支持失败重试。
- 自动任务必须记录日志。
- 生成任务可以取消或重新生成。
- 页面刷新后可以恢复任务状态。

### SSE 可靠性

- SSE 断开后，前端可以重新连接并查询任务结果。
- 生成过程中的阶段结果需要落库。
- LLM 原始输出和结构化结果需要保留。
- 错误信息需要返回给前端并写入任务日志。

### 安全性

- API 密钥不得暴露在前端。
- 后台必须登录访问。
- 表单输入需要做长度限制和基础校验。
- 删除和发布状态变更需要记录操作者。

### 易用性

- 后台页面面向非开发人员设计。
- 所有主要按钮、表单项、状态和错误提示必须使用业务语言。
- 普通运营界面不展示接口字段、模型参数、提示词变量或 JSON 结构。
- 默认值要覆盖大多数使用场景，运营只需填写内容意图和生成数量。
- 生成失败时要提示下一步动作，例如“换一个更具体的主题后重试”。

### 内容安全

- 不生成违法、歧视、侵权、低俗内容。
- 不输出医疗、金融、法律等高风险定论型建议。
- 不做虚假承诺或夸张收益表达。
- 生成内容必须经过人工审核后才允许合成和导出。

### 可扩展性

- LLM 调用层可替换。
- 视频模板可新增。
- 栏目模板可新增。
- 发布渠道可扩展。
- 后续可再增加公开前台，但不属于当前阶段。

## 14. MVP 范围

MVP 必做：

1. 后台登录。
2. 内置 1 个内容方向和 3 个默认栏目。
3. 提供语义化内容生成表单。
4. 根据输入方向抓取网络相关内容。
5. 形成素材汇总和来源列表。
6. 调用 LLM 总结生成选题、脚本、分镜和字幕。
7. 使用 SSE 在后台实时显示抓取、汇总和生成过程。
8. 提供话题监控和每日自动抓取任务。
9. 提供单列表审核后台。
10. 合成竖屏视频。
11. 生成封面和知识卡片。
12. 导出发布包。
13. 回填外部发布链接。
14. 提供基础数据看板。
15. 生成每日产出报告。

MVP 验收：

1. 非开发人员输入“想做什么内容”后，系统可以先抓取相关网络内容。
2. 每次生成任务可以形成素材汇总和来源列表。
3. 基于素材汇总可以生成不少于 10 个候选选题。
4. 选择一个选题后，可以通过 SSE 流式生成 60 秒脚本。
5. 脚本生成后，可以通过 SSE 流式生成分镜和字幕。
6. 可以为某个话题开启监控，并每天自动形成素材汇总和内容列表。
7. 后台可以审核、修改、驳回和重新生成内容。
8. 审核通过后，可以合成不少于 3 条竖屏视频。
9. 可以导出包含视频、封面、标题、简介、标签和口播稿的发布包。
10. 可以回填外部平台发布链接。
11. 后台可查看今日产出、抓取状态、任务状态和失败原因。
12. 每日报告可查看或导出。
13. 系统不包含公开前台网站和公开内容页。
14. 普通运营界面不出现 `prompt`、`model`、`temperature`、`task_id` 等技术字段。

## 19. CEO 汇报口径

当前阶段先建设后台内容生产系统，不做公开前台网站。运营人员在后台输入内容方向或主题，系统先抓取相关网络内容并形成素材汇总，再调用 LLM 自动生成选题、脚本、分镜、字幕和素材文案，并通过 SSE 实时展示抓取、汇总和生成过程。

项目价值先通过后台生产效率验证：每天能抓取和汇总多少素材，能生成多少选题、脚本和视频素材，审核通过率如何，视频合成是否稳定，发布包是否能支撑运营人员手动发布。公开网站、订阅、线索收集和 SEO 可作为后续阶段扩展。
