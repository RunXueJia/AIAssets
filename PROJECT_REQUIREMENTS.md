# 24 小时 AI 增长资产引擎项目需求说明书

版本：v1.0  
日期：2026-05-18  
项目方向：短视频 / 科普视频 / 职场效率内容资产  
项目定位：对外运行的 AI 内容增长系统，不接入公司内部业务系统，不以监控预警为核心。

## 1. 项目概述

本项目建设一个 24 小时自动运行的对外内容增长资产系统，围绕“AI 工具科普、职场效率、短视频知识内容”垂直方向，自动完成选题、脚本、分镜、字幕、视频合成、图文页面、资料包和发布队列的生产。

系统的核心不是单纯批量生成短视频，而是持续沉淀可复用的外部增长资产：

- 短视频内容资产
- 图文科普页面
- 知识卡片
- 下载资料包
- 内容专题页
- 用户注册和线索
- 数据化内容运营后台

项目第一阶段以“AI 工具 + 职场效率科普”为垂直内容方向，面向普通职场人、学生、小微企业主、自媒体运营人员，提供可理解、可执行、可复用的科普内容。

## 2. 立项目标

### 2.1 业务目标

1. 建立一个可 24 小时自动运行的对外内容生产系统。
2. 每天稳定产出短视频、图文页、知识卡片和资料包。
3. 通过公开网站和平台账号沉淀外部流量。
4. 通过免费工具、资料下载、订阅和表单收集潜在用户线索。
5. 用可量化指标向公司证明项目价值，而不是只展示 AI Demo。

### 2.2 产品目标

1. 运营人员输入一个垂直主题后，系统可以自动生成一批可审核的内容任务。
2. 编辑人员可以在后台审核、修改、通过或驳回内容。
3. 系统可以把通过审核的脚本自动合成为竖屏短视频。
4. 每条视频自动生成对应的公开图文页面。
5. 系统每日生成产出报表，展示内容数量、发布状态、访问量和转化数据。

### 2.3 技术目标

1. 任务调度稳定运行，支持 24 小时自动执行。
2. 内容生产链路可追踪、可重试、可暂停。
3. 生成内容有状态管理和版本记录。
4. 支持模型、视频模板、栏目模板和发布渠道的后续扩展。
5. 不依赖公司内部业务数据库，不读取公司内部敏感资料。

## 3. CEO 视角价值

该项目对公司的价值不是“自动生成视频”，而是建立一个对外增长机器：

1. 用内容获取自然流量。
2. 用网站沉淀搜索资产。
3. 用资料下载收集潜在用户。
4. 用后台数据判断哪些主题值得继续投入。
5. 用自动化降低内容团队的边际生产成本。

项目可向管理层汇报的核心结果：

- 今日新增短视频数量
- 今日新增图文页面数量
- 今日新增知识卡片数量
- 今日新增资料包数量
- 今日发布内容数量
- 今日网站访问量
- 今日注册 / 下载 / 留资数量
- 本周表现最好的主题
- 本周内容资产累计规模

## 7. 核心业务流程

### 7.1 自动生产流程

1. 系统根据内容垂类和栏目配置生成候选选题。
2. 系统为每个选题生成标题、关键词、目标受众和内容角度。
3. 系统生成短视频脚本，包括开头钩子、主体内容、结尾引导。
4. 系统生成分镜，包括画面描述、字幕、旁白和素材建议。
5. 系统进行基础事实检查和风险词检查。
6. 内容进入待审核状态。
7. 编辑审核并修改内容。
8. 审核通过后进入视频合成队列。
9. 系统按模板合成竖屏视频。
10. 系统生成封面、标题、简介、标签和平台适配文案。
11. 系统生成对应图文页面。
12. 系统生成知识卡片和资料包。
13. 内容进入发布队列。
14. 运营人员发布或导出。
15. 系统统计内容产出、访问、下载和转化数据。

### 7.2 人工审核流程

1. 自动生成内容后，状态为“待审核”。
2. 编辑可以查看脚本、分镜、字幕、来源说明和风险提示。
3. 编辑可以执行“通过”“驳回”“修改后通过”“重新生成”。
4. 驳回内容必须填写原因，便于后续优化提示词。
5. 修改后通过的内容需要保存版本记录。
6. 未审核内容不得自动发布到外部平台。

### 7.3 每日运行流程

默认每日自动任务：

- 00:30 生成新选题
- 01:00 生成脚本
- 02:00 生成分镜和字幕
- 03:00 执行质量检查
- 04:00 生成图文页面草稿
- 05:00 生成知识卡片草稿
- 06:00 生成资料包草稿
- 09:00 推送待审核列表
- 12:00 合成已审核视频
- 18:00 生成当日产出日报
- 23:30 清理失败任务并重试

具体时间可在后台配置。

## 8. 功能需求

### 8.1 内容方向配置

后台应支持配置：

- 垂直领域
- 栏目名称
- 目标用户
- 内容风格
- 禁止话题
- 平台限制
- 每日目标产量
- 审核人
- 默认视频模板
- 默认图文模板

验收标准：

- 运营人员可以创建、编辑、启用、停用一个内容方向。
- 每个内容方向可以绑定多个栏目。
- 停用后的方向不再产生新任务。

### 8.2 选题生成

系统根据栏目配置自动生成选题。

每个选题包含：

- 标题
- 副标题
- 栏目
- 目标受众
- 内容角度
- 关键词
- 难度等级
- 预计视频时长
- 推荐发布平台
- 生成原因

验收标准：

- 每日可按配置生成不少于 50 个候选选题。
- 选题重复率需要控制在 10% 以下。
- 选题必须能进入审核、驳回、锁定和生成脚本流程。

### 8.3 脚本生成

系统为选题生成短视频脚本。

脚本结构：

- 3 秒开头钩子
- 问题场景
- 核心方法
- 操作步骤
- 示例
- 结尾引导
- 平台标题
- 平台简介
- 标签

支持脚本版本：

- 15 秒极简版
- 60 秒标准版
- 180 秒深度版

验收标准：

- 每个选题至少生成 1 个 60 秒标准版脚本。
- 脚本必须分段，便于生成字幕和分镜。
- 脚本需要标记可疑事实、敏感表述和夸张承诺。

### 8.4 分镜生成

系统将脚本拆成视频分镜。

每个分镜包含：

- 镜头序号
- 时长
- 旁白
- 屏幕字幕
- 画面类型
- 素材建议
- 动效建议
- 背景音乐情绪

画面类型包括：

- 标题页
- 要点列表
- 工具界面演示
- 对比画面
- 流程图
- 案例展示
- 总结页
- 关注引导页

验收标准：

- 60 秒视频建议生成 6 到 10 个分镜。
- 分镜总时长与脚本时长误差不超过 10 秒。
- 每个分镜必须有字幕和画面建议。

### 8.5 视频合成

系统根据分镜和模板合成竖屏短视频。

第一阶段视频规格：

- 比例：9:16
- 分辨率：1080x1920
- 格式：MP4
- 帧率：30fps
- 单条时长：15 秒、60 秒或 180 秒
- 字幕：硬字幕
- 封面：自动生成

模板类型：

- 知识讲解模板
- 工具步骤模板
- 对比分析模板
- 清单总结模板
- 问答解释模板

验收标准：

- 已审核脚本可以进入视频合成队列。
- 合成失败需要记录原因并支持重试。
- 合成后可以在线播放、下载和重新生成。
- 字幕不能遮挡主要内容。
- 标题和字幕不能超出画面边界。

### 8.6 图文页面生成

每条通过审核的视频应自动生成一个公开图文页面。

页面内容：

- 页面标题
- 摘要
- 视频播放器
- 文字稿
- 操作步骤
- 使用场景
- 注意事项
- 相关工具
- 相关内容推荐
- 资料下载入口
- 订阅 / 留资表单

验收标准：

- 每条发布内容都有唯一 URL。
- 页面支持搜索引擎收录。
- 页面可以关联视频、资料包和知识卡片。
- 页面可配置是否公开。

### 8.7 知识卡片生成

系统从脚本中提取核心知识点生成图片卡片。

卡片类型：

- 一句话总结
- 三步操作法
- 常见误区
- 工具清单
- 对比表
- 流程图

验收标准：

- 每条标准视频至少生成 3 张知识卡片。
- 卡片可以下载为 PNG。
- 卡片文字必须完整可读。
- 卡片应使用统一品牌样式。

### 8.8 资料包生成

系统按主题生成可下载资料包。

资料包类型：

- PDF 指南
- Prompt 模板
- Excel 清单
- Notion 模板说明
- PPT 大纲
- 检查表

验收标准：

- 资料包必须与视频主题相关。
- 用户下载资料包前可以要求填写邮箱、微信或手机号。
- 系统需要记录下载行为。
- 资料包内容不能包含未经授权的第三方材料。

### 8.9 发布队列

系统提供内容发布管理能力。

发布状态：

- 草稿
- 待审核
- 审核通过
- 合成中
- 合成失败
- 待发布
- 已发布
- 已下线

发布渠道：

- 官网
- 抖音
- 小红书
- B站
- 视频号
- YouTube Shorts
- TikTok

第一阶段可先支持“导出发布资料”，不强制直连平台 API。

导出内容包括：

- 视频文件
- 封面图
- 标题
- 简介
- 标签
- 评论区置顶文案
- 图文版内容

验收标准：

- 运营人员可以筛选待发布内容。
- 运营人员可以下载发布包。
- 已发布内容可以记录平台链接。
- 后台可以按平台查看发布状态。

### 8.10 公开网站

公开网站用于承接外部访问和搜索流量。

页面类型：

- 首页
- 栏目页
- 视频详情页
- 图文详情页
- 专题页
- 资料下载页
- 工具推荐页
- 订阅页

首页应展示：

- 最新视频
- 热门栏目
- 热门资料包
- 推荐专题
- 订阅入口

验收标准：

- 网站无需登录即可浏览公开内容。
- 页面加载速度应满足基本 SEO 要求。
- 页面支持移动端浏览。
- 每个内容页面有清晰的转化入口。

### 8.11 线索收集

系统应提供轻量级线索收集能力。

线索来源：

- 资料包下载
- 邮件订阅
- 表单咨询
- 工具试用申请
- 企业培训咨询

线索字段：

- 姓名
- 联系方式
- 公司名称
- 职位
- 需求类型
- 来源页面
- 提交时间

验收标准：

- 表单提交成功后进入线索列表。
- 线索可以导出 CSV。
- 线索需要记录来源内容。
- 表单必须有用户授权提示。

### 8.12 数据看板

后台提供基础指标看板。

核心指标：

- 今日生成选题数
- 今日生成脚本数
- 今日合成视频数
- 今日新增图文页数
- 今日新增知识卡片数
- 今日新增资料包数
- 今日发布数量
- 今日访问量
- 今日下载量
- 今日线索数
- 任务成功率
- 任务失败率

验收标准：

- 管理层可以快速看到今日产出。
- 运营可以查看不同栏目表现。
- 技术管理员可以查看任务失败原因。

### 8.13 每日报告

系统每天自动生成项目日报。

日报内容：

- 今日产出总览
- 各栏目产出数量
- 已发布内容列表
- 待审核内容列表
- 失败任务列表
- 网站访问数据
- 下载和线索数据
- 明日计划建议

验收标准：

- 每天固定时间生成日报。
- 日报可在后台查看。
- 日报支持导出 Markdown 或 PDF。
- 日报可以推送到邮箱、飞书或企业微信，第一阶段可先支持后台查看和手动导出。

## 9. 非功能需求

### 9.1 稳定性

- 系统需要支持 24 小时运行。
- 单个任务失败不能阻塞整个任务队列。
- 任务需要支持失败重试。
- 所有自动任务需要记录日志。
- 系统应支持手动暂停和恢复任务。

### 9.2 安全性

- API 密钥不能明文展示在前端。
- 后台需要账号登录。
- 管理员、编辑、运营角色权限分离。
- 公开页面不得暴露内部配置。
- 用户表单数据需要基础访问控制。

### 9.3 内容安全

- 生成内容不得含有明显违法、歧视、侵权、低俗内容。
- 不输出医疗、金融、法律等高风险定论型建议。
- 涉及工具推荐时需要避免虚假承诺。
- 涉及 AI 生成内容时，页面可保留“内容由 AI 辅助生成并经人工审核”的说明。
- 所有对外发布内容必须经过人工审核。

### 9.4 可扩展性

- 模型调用层需要可替换。
- 视频模板需要可新增。
- 栏目模板需要可新增。
- 发布渠道需要可扩展。
- 内容类型需要支持后续新增音频、长视频、直播脚本等形式。

### 9.5 可维护性

- 任务状态清晰可追踪。
- 关键配置后台可维护。
- 日志包含任务 ID、内容 ID、错误原因和重试次数。
- 生成内容保留版本记录。

## 11. 推荐技术架构

### 11.1 系统模块

1. Web 前台  
   展示公开内容、视频、资料包和表单。

2. Admin 后台  
   管理内容方向、审核、发布队列、数据看板。

3. AI 生成服务  
   负责选题、脚本、分镜、图文、资料包的生成。

4. 视频渲染服务  
   负责按模板合成视频和封面。

5. 任务调度服务  
   负责定时任务、队列、失败重试。

6. 数据服务  
   负责内容、资产、任务、线索和指标数据存储。

7. 文件存储服务  
   存储视频、封面、卡片、PDF 和导出包。

### 11.2 技术选型建议

第一阶段建议选简单稳定的组合：

- 前端：Next.js 或 Vue
- 后台管理：Next.js Admin / Vue Admin
- 后端：FastAPI / NestJS
- 数据库：PostgreSQL
- 队列：Redis + BullMQ / Celery
- 定时任务：Cron / APScheduler
- 文件存储：本地 MinIO 或云对象存储
- 视频合成：FFmpeg + HTML/CSS 模板渲染
- AI 模型：可配置 OpenAI 兼容接口或本地模型接口

### 11.3 部署建议

最小部署：

- 1 台应用服务器
- 1 个 PostgreSQL 数据库
- 1 个 Redis
- 1 个对象存储目录或 MinIO
- 1 个反向代理 Nginx

生产部署建议：

- Web 服务和任务服务分离
- 视频渲染服务单独部署
- 数据库定期备份
- 文件存储定期备份
- 日志集中保存

## 16. 内容质量标准

### 16.1 合格内容标准

- 主题清晰
- 标题不夸张
- 开头 3 秒有明确问题或收益
- 观点具体
- 步骤可执行
- 没有明显事实错误
- 字幕可读
- 画面不空洞
- 结尾有合理引导

### 16.2 不合格内容标准

- 标题党
- 空泛鸡汤
- 信息重复
- 没有实际步骤
- AI 味过重
- 事实不确定却用肯定语气
- 承诺收益过高
- 涉及高风险建议
- 画面模板过于重复

### 16.3 人工审核重点

- 是否有事实错误
- 是否有误导性承诺
- 是否侵犯第三方版权
- 是否适合公开发布
- 是否符合栏目定位
- 是否需要补充免责声明

## 17. 风险与应对

### 17.1 内容同质化风险

风险：批量 AI 内容容易重复、空泛。  
应对：设置栏目模板、案例库、表达风格、人工审核和重复检测。

### 17.2 平台限制风险

风险：平台对低质量 AI 内容和搬运内容更加严格。  
应对：坚持原创脚本、人工审核、统一品牌表达，不做无审核自动发布。

### 17.3 事实错误风险

风险：AI 可能编造工具功能、数据和结论。  
应对：对事实型内容进行来源检查，对不确定内容使用谨慎表达。

### 17.4 版权风险

风险：素材、音乐、图片或第三方界面可能存在版权问题。  
应对：使用自有模板、可商用素材库、开源字体和授权音乐。

### 17.5 技术成本风险

风险：视频生成和模型调用成本过高。  
应对：先使用模板化视频，限制每日产量，按审核后再合成视频。

### 17.6 运营无人接手风险

风险：系统能产出，但没有人审核和发布。  
应对：MVP 阶段明确每天固定审核窗口，日报中列出待处理任务。
